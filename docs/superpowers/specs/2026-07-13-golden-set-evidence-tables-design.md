# Golden Set 冻结证据表格化设计

## 目标

改进 ProductConfig Golden Set v2 全文盲审入口 `/agent/golden-set/full-review` 的冻结证据展示。标注员应直接看懂 Excel 配置、产品候选和 ERP 候选，不需要理解 `[SEL]`、`[ ]`、`option_set` 或原始 JSON。

本次只改变前端派生展示。冻结 evidence 内容、evidence hash、后端 API、草稿与提交结构、A/B 盲标隔离及校验规则保持不变。

## 已确认方案

采用按业务含义分组的证据展示：

1. Excel 配置证据显示为“配置项 / 可选内容”两列表格。
2. 产品候选显示为“来源 / 产品名称”两列表格。
3. ERP 候选显示为“公司与物料号 / 产品信息”两列表格。
4. Excel 选项统一使用只读复选框：框内打勾表示选中，空框表示未选中。页面不显示 `[SEL]`、`[ ]` 或“未选”文字。
5. 单元格坐标、BOM 等追溯信息放在主值下方作为次要信息，避免增加表格列数。
6. 原始 evidence 始终保留在“查看原始证据”折叠区，解析失败时不丢失内容。

未采用“只显示已选项”，因为标注员仍需看到被排除的备选项；未采用 Excel 原表复刻，因为不同模板的列结构不稳定且手机端容易拥挤。

## 前端数据映射

`ChineseEvidenceCard` 根据 `evidence_id` 选择纯前端解析路径：

- `block:*`：解析 `Sheet`、`Row`、`[A1]` 等单元格行和紧随其后的 `option_set`。结构化 `option_set.options[].selected` 是复选框状态的首选来源；仅在没有结构化选项块时，才用 `[SEL]` / `[ ]` 推断。
- `package-candidates:*`：解析 JSON 数组中的 `source` 与 `value`，生成产品候选表格。
- `erp-candidates:*`：解析 JSON 对象中的 `status`、`reason` 与 `candidates`，生成 ERP 候选表格；无候选或查询未解决时显示明确中文状态，不伪造候选行。

Excel 行以业务字段为单位呈现。字段名优先使用 `option_set.field`；没有字段名时，使用同一行前置单元格文本；仍无法判断时使用坐标作为字段名。普通非选项行按同一 Row 的单元格顺序组合为字段和值，避免把每个单元格拆成独立卡片。

解析函数只生成展示模型，不改写或回传原始 evidence。未知字段保持原值，不因前端展示规则过滤业务内容。

## 组件与样式

- `types.ts` 增加证据分组、表格行和只读选项的展示类型。
- `utils.ts` 集中处理三类 evidence 的解析、中文状态和安全回退，保持纯函数并通过现有 Node 测试验证。
- `ChineseEvidenceCard.tsx` 按证据分组渲染表格、只读复选框和回退内容。
- `styles.css` 增加证据分组、两列表格、复选框与响应式样式，不引入新依赖。

桌面端证据表格固定为两列，不为坐标、BOM 或状态再增加列。中文主值避免按单字强制断行，长英文料号允许安全换行。

在 900px 以下，表头隐藏，每一行改为“字段说明在上、内容在下”的纵向布局。360px、390px、430px 下不得出现横向滚动、中文逐字换行或底部操作栏遮挡。

## 错误与回退

- JSON 无法解析、结构不是预期类型或 Excel 行无法归组时，该 evidence 显示“暂时无法结构化展示”，并引导在下方展开原始证据。
- 解析单条候选失败不得影响其他 evidence 分组。
- ERP `unresolved` 原因转换为简明中文；未知原因显示通用“暂未取得 ERP 候选”，原始原因仍可在折叠区核对。
- 原始证据继续使用 `<pre>` 保留换行和追溯信息，不修改冻结内容。

## 验证

扩展 `goldenSetFullReviewUiMapping.test.ts`，覆盖：

- `option_set` 的选中与空框状态；
- 没有 `option_set` 时的 `[SEL]` / `[ ]` 回退；
- Excel Row 与单元格归组；
- 产品候选与 ERP 候选 JSON 映射；
- malformed JSON 和未知结构的安全回退；
- 输出展示模型不包含需要直接呈现的 `[SEL]` 说明文本。

运行专项 Node 测试和 `npm run build:web`。使用浏览器打开真实页面，分别检查桌面、360px、390px、430px，并确认三类表格、复选框、原始证据折叠区和标注表单交互正常。

## 非目标

- 不修改 Excel parser 的标准化文本格式。
- 不重建或覆盖 v2 sealed packet。
- 不修改证据 hash、API 契约、数据库、ERP 查询或归档流程。
- 不增加可编辑复选框；证据区复选框仅表示原始配置状态。
